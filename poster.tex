% ----------------------------------
% Abstract
% ----------------------------------
  \begin{block}{Abstract}
    Traffic signal control in urban environments must balance vehicle throughput
    with pedestrian safety --- a requirement that classical rule-based methods
    cannot meet simultaneously. We present an \textbf{Action-Constrained
    Proximal Policy Optimisation (PPO)} agent that learns adaptive signal timing
    while enforcing hard pedestrian safety constraints through feasible-action
    masking at every decision step. The agent is evaluated against five
    baselines --- Fixed-Time, Max Pressure, SOTL, DQN-AR, and SPRe+ --- on a
    single-intersection toy task and a 3$\times$3 grid network simulated in
    SUMO. The proposed PPO agent achieves the \textbf{lowest waiting time among
    all safe agents} and \textbf{zero constraint violations} in both scenarios,
    while all rule-based baselines violate safety constraints in 19--71\% of
    steps.
  \end{block}

% ----------------------------------
% Research Objectives
% ----------------------------------
  \begin{alertblock}{Research Objectives}
    \begin{itemize}
      \item \textbf{Objective 1:} Design an action-constrained RL agent that
        enforces pedestrian safety constraints \emph{by construction}, guaranteeing
        zero violations without relying on reward shaping alone.
      \item \textbf{Objective 2:} Benchmark the proposed agent against
        rule-based baselines (Fixed-Time, Max Pressure, SOTL) and
        action-constrained learned agents (DQN-AR, SPRe+) on efficiency and
        safety metrics across two network scales.
      \item \textbf{Objective 3:} Validate the computational advantage of
        acceptance-rejection masking over QP-projection (SPRe+) in growing
        action spaces.
    \end{itemize}
  \end{alertblock}

% ----------------------------------
% Methodology
% ----------------------------------
  \begin{block}{Methodology}

    The agent operates within an \textbf{Action-Constrained MDP}.  At every
    step, the constraint checker computes the feasible action set
    $\mathcal{C}(s)$ by removing phases that violate any hard constraint.  PPO
    selects actions via a \textbf{masked softmax} restricted to
    $\mathcal{C}(s)$, so infeasible actions receive zero probability and can
    never be executed.

    \vspace{0.6cm}
    \begin{center}
    \begin{tikzpicture}[node distance=2.4cm]
      \node (sumo) [startstop]
        {\textbf{SUMO Simulation}\\
         Vehicle \& pedestrian arrivals (Poisson process)};
      \node (obs)  [process, below of=sumo]
        {\textbf{State} $s_t$ (31-dim): phase timer, vehicle density,
         vehicle wait, pedestrian wait, crossing occupancy};
      \node (ppo)  [process, below of=obs]
        {\textbf{PPO Policy} $\pi_\theta$: two-layer MLP,
         hidden dim 128, masked softmax over $\mathcal{C}(s)$};
      \node (mask) [process, below of=ppo]
        {\textbf{Constraint Checker}: yellow lock $\cdot$ min green 10\,s
         $\cdot$ max green 60\,s $\cdot$ pedestrian safety};
      \node (act)  [startstop, below of=mask]
        {\textbf{Action Execution}: selected phase $\rightarrow$ SUMO via TraCI,
         reward $r_t = -(\alpha W_t^v + \beta W_t^p)$ computed};
      \draw [arrow] (sumo) -- (obs);
      \draw [arrow] (obs)  -- (ppo);
      \draw [arrow] (ppo)  -- (mask);
      \draw [arrow] (mask) -- (act);
      \draw [arrow] (act.east) -- ++(4cm,0) |- (sumo.east);
    \end{tikzpicture}
    \end{center}
  \end{block}

% ----------------------------------
% Agents
% ----------------------------------
  \begin{block}{Agents \& Baselines}
    \begin{itemize}
      \item \textbf{Fixed-Time:} Alternates NS/EW on a fixed 30\,s cycle.
        No traffic awareness. Lower-bound safety reference.
      \item \textbf{Max Pressure \cite{varaiya2013}:} Selects the phase
        maximising total queue pressure. Vehicle-centric; no pedestrian safety.
      \item \textbf{SOTL \cite{gershenson2005}:} Switches when red-side queue
        $\Phi_\text{red}(t)\!\geq\!\kappa\!=\!5$ and $\tau\!\geq\!\tau_\text{min}$.
        Decentralised and adaptive.
      \item \textbf{DQN-AR:} Deep Q-Network with acceptance-rejection safety
        filter. Zero violations by resampling until a feasible action is drawn.
        $\mathcal{O}(1)$ per-step cost.
      \item \textbf{SPRe+ \cite{hung2025}:} Projects the full policy
        distribution onto the feasible simplex via quadratic programming.
        $\mathcal{O}(|\mathcal{A}|^2)$ per step; equivalent to DQN-AR in binary
        action spaces.
      \item \textbf{Action-Constrained PPO (proposed):} PPO with masked
        softmax over $\mathcal{C}(s)$. Learns safe behaviour during training
        rather than enforcing it post-hoc.
    \end{itemize}
  \end{block}

    \vspace{0.8cm}
    \heading{Key Findings}
    \begin{itemize}
      \item PPO reduces average vehicle wait by \textbf{44\%} vs.\ Fixed-Time
        (1$\times$1) and \textbf{20\%} vs.\ Fixed-Time (3$\times$3).
      \item The only agent with lower wait than PPO in 1$\times$1 (Max
        Pressure, 55.6\,s) violates safety constraints in \textbf{53.5\% of
        steps} and is unsuitable for real deployment.
      \item All three learned agents (PPO, DQN-AR, SPRe+) achieve
        \textbf{exactly 0\% violations}; all rule-based baselines violate
        constraints in \textbf{19--71\% of steps}.
      \item PPO's advantage is larger in 1$\times$1 ($-$16\% vs.\ SOTL) than
        in 3$\times$3 ($-$6.4\% vs.\ Max Pressure), consistent with the
        increased difficulty of a partially-controlled 9-intersection network.
    \end{itemize}

% ----------------------------------
% Conclusions
% ----------------------------------
  \begin{block}{Conclusions}
    \begin{itemize}
      \item Action-constrained PPO achieves the \textbf{best safety--efficiency
        trade-off} in both scenarios: lowest waiting time among safe agents and
        zero constraint violations by construction.
      \item Feasible-action masking during training eliminates the need for
        post-hoc projection or violation penalties used by SPRe+ and DQN-AR.
      \item DQN-AR and SPRe+ produce identical results in this study because
        the 4-phase action space is small; they would diverge in larger
        networks where QP cost $\mathcal{O}(|\mathcal{A}|^2)$ becomes a
        bottleneck.
      \item 100 training episodes suffice for the 1$\times$1 task; the 3$\times$3
        policy had not fully converged, indicating further improvement is
        achievable with longer training.
      \item Full 9-intersection multi-agent control and statistical evaluation
        over multiple random seeds remain as primary directions for future work.
    \end{itemize}
  \end{block}

% ----------------------------------
% Related Work
% ----------------------------------
  \begin{exampleblock}{What Is Already Known About This Subject?}
    \begin{itemize}
      \item \textbf{Max Pressure \cite{varaiya2013}} provides a
        throughput-optimal, non-learning upper bound for vehicle metrics,
        but it is vehicle-centric and contains no pedestrian safety mechanism.
      \item \textbf{SOTL \cite{gershenson2005}} demonstrates effective
        decentralised reactive control without coordination or training, widely
        used as a benchmark heuristic in traffic RL literature.
      \item \textbf{SPRe+ \cite{hung2025}} is the state-of-the-art
        action-constrained RL method via QP projection; it guarantees zero
        violations but incurs quadratic per-step cost as action space grows.
    \end{itemize}
  \end{exampleblock}

% ----------------------------------
% Practical Implications
% ----------------------------------
  \begin{exampleblock}{Practical Implications}
    \begin{itemize}
      \item \textbf{Hardware compatibility:} Zero constraint violations mean the
        agent never requests an illegal phase transition --- directly compatible
        with real intersection controllers enforcing minimum green times.
      \item \textbf{Scalability:} Feasible-action masking scales linearly with
        the number of phases, unlike QP-based methods whose cost grows
        quadratically.
      \item \textbf{Pedestrian safety guarantee:} Incorporating pedestrian state
        into $\mathcal{C}(s)$ provides a structural safety guarantee that
        reward shaping or post-hoc filtering alone cannot match.
    \end{itemize}
  \end{exampleblock}
