% ----------------------------------
% Abstract
% ----------------------------------
  \begin{block}{Abstract}
    Traffic signal control in urban environments must balance vehicle throughput
    with pedestrian safety --- a requirement that classical rule-based methods
    cannot meet simultaneously. We present an \textbf{Action-Constrained
    Proximal Policy Optimisation (PPO)} agent that learns adaptive signal timing
    while enforcing hard pedestrian safety constraints through feasible-action
    masking at every decision step. The agent is evaluated against five
    baselines --- Fixed-Time, Max Pressure, SOTL, DQN-AR, and SPRe+ --- on a
    single-intersection toy task and a 3$\times$3 grid network simulated in
    SUMO. The proposed PPO agent achieves the \textbf{lowest waiting time among
    all safe agents} and \textbf{zero constraint violations} in both scenarios,
    while all rule-based baselines violate safety constraints in 19--71\% of
    steps.
  \end{block}

% ----------------------------------
% Research Objectives
% ----------------------------------
  \begin{alertblock}{Research Objectives}
    \begin{itemize}
      \item \textbf{Objective 1:} Design an action-constrained RL agent that
        enforces pedestrian safety constraints \emph{by construction},
        guaranteeing zero violations without relying on reward shaping alone.
      \item \textbf{Objective 2:} Benchmark the proposed agent against
        rule-based baselines (Fixed-Time, Max Pressure, SOTL) and
        action-constrained learned agents (DQN-AR, SPRe+) on efficiency and
        safety metrics across two network scales.
      \item \textbf{Objective 3:} Validate the computational advantage of
        acceptance-rejection masking over QP-projection (SPRe+) in growing
        action spaces.
    \end{itemize}
  \end{alertblock}

% ----------------------------------
% Methodology
% ----------------------------------
  \begin{block}{Methodology}

    The agent operates within an \textbf{Action-Constrained MDP}. At every
    step, the constraint checker computes the feasible action set
    $\mathcal{C}(s)$ by removing phases that violate any hard constraint. PPO
    selects actions via a \textbf{masked softmax} restricted to $\mathcal{C}(s)$,
    so infeasible actions receive zero probability and can never be executed.

    \vspace{0.6cm}
    \begin{center}
    \begin{tikzpicture}[node distance=2.4cm]
      \node (sumo) [startstop]
        {\textbf{SUMO Simulation}\\
         Vehicle \& pedestrian arrivals (Poisson process)};
      \node (obs)  [process, below of=sumo]
        {\textbf{State} $s_t$ (31-dim): phase timer, vehicle density,
         vehicle wait, pedestrian wait, crossing occupancy};
      \node (ppo)  [process, below of=obs]
        {\textbf{PPO Policy} $\pi_\theta$: two-layer MLP,
         hidden dim 128, masked softmax over $\mathcal{C}(s)$};
      \node (mask) [process, below of=ppo]
        {\textbf{Constraint Checker}: yellow lock $\cdot$ min green 10\,s
         $\cdot$ max green 60\,s $\cdot$ pedestrian safety};
      \node (act)  [startstop, below of=mask]
        {\textbf{Action Execution}: selected phase $\to$ SUMO via TraCI,
         reward $r_t = -(\alpha W_t^v + \beta W_t^p)$ computed};
      \draw [arrow] (sumo) -- (obs);
      \draw [arrow] (obs)  -- (ppo);
      \draw [arrow] (ppo)  -- (mask);
      \draw [arrow] (mask) -- (act);
      \draw [arrow] (act.east) -- ++(4cm,0) |- (sumo.east);
    \end{tikzpicture}
    \end{center}

  \end{block}

% ----------------------------------
% Agents & Baselines
% ----------------------------------
  \begin{block}{Agents \& Baselines}
    \begin{itemize}
      \item \textbf{Fixed-Time:} Fixed 30\,s NS/EW cycle. No traffic awareness.
      \item \textbf{Max Pressure:} Selects phase maximising
        queue pressure. Vehicle-centric; no pedestrian safety.
      \item \textbf{SOTL:} Switches when red-side queue
        $\Phi_\text{red}(t)\!\geq\!\kappa\!=\!5$ and $\tau\!\geq\!\tau_\text{min}$.
      \item \textbf{DQN-AR:} DQN with acceptance-rejection safety filter.
        Zero violations; $\mathcal{O}(1)$ per-step cost.
      \item \textbf{SPRe+ :} Projects policy onto feasible simplex
        via QP. $\mathcal{O}(|\mathcal{A}|^2)$; equivalent to DQN-AR here.
      \item \textbf{Action-Constrained PPO (proposed):} Masked softmax over
        $\mathcal{C}(s)$. Learns safe behaviour during training.
    \end{itemize}
  \end{block}

    \vspace{0.6cm}
    \heading{Key Findings}
    \begin{itemize}
      \item PPO reduces average vehicle wait by \textbf{44\%} vs.\ Fixed-Time
        (1$\times$1) and \textbf{20\%} vs.\ Fixed-Time (3$\times$3).
      \item The only agent with lower wait than PPO in 1$\times$1 (Max Pressure,
        55.6\,s) violates safety in \textbf{53.5\% of steps} --- unsuitable for
        deployment.
      \item All three learned agents achieve \textbf{0\% violations}; all
        rule-based baselines violate in \textbf{19--71\% of steps}.
      \item PPO's advantage shrinks in 3$\times$3 ($-$6.4\% vs.\ Max Pressure)
        vs.\ 1$\times$1 ($-$16\% vs.\ SOTL), consistent with a
        partially-controlled 9-intersection network.
    \end{itemize}

% ----------------------------------
% Conclusions
% ----------------------------------
  \begin{block}{Conclusions}
    \begin{itemize}
      \item Action-constrained PPO achieves the \textbf{best safety--efficiency
        trade-off}: lowest waiting time among safe agents, zero violations by
        construction.
      \item Feasible-action masking during training eliminates the post-hoc
        projection overhead of SPRe+ and the resampling loop of DQN-AR.
      \item DQN-AR and SPRe+ yield identical results here (small action space);
        they would diverge in larger networks where QP cost becomes a bottleneck.
      \item 100 episodes suffice for 1$\times$1; the 3$\times$3 policy had not
        fully converged --- longer training would widen the gap over baselines.
      \item Future work: full 9-intersection multi-agent control, multi-seed
        statistical evaluation, transfer to real intersection data.
    \end{itemize}
  \end{block}

% ----------------------------------
% Related Work
% ----------------------------------
  \begin{exampleblock}{What Is Already Known About This Subject?}
    \begin{itemize}
      \item \textbf{Max Pressure} provides a throughput-optimal
        non-learning upper bound but is vehicle-centric with no pedestrian safety.
      \item \textbf{SOTL} achieves effective decentralised
        reactive control without training, widely used as a benchmark heuristic.
      \item \textbf{SPRe+ } is the state-of-the-art
        action-constrained RL method via QP projection; guarantees zero violations
        but incurs quadratic per-step cost as the action space grows.
    \end{itemize}
  \end{exampleblock}

% ----------------------------------
% Practical Implications
% ----------------------------------
  \begin{exampleblock}{Practical Implications}
    \begin{itemize}
      \item \textbf{Hardware compatibility:} Zero violations mean the agent never
        requests an illegal phase transition --- directly compatible with real
        controllers enforcing minimum green times.
      \item \textbf{Scalability:} Feasible-action masking scales linearly with
        the number of phases, unlike QP-based methods (quadratic cost).
      \item \textbf{Pedestrian safety guarantee:} Incorporating pedestrian state
        into $\mathcal{C}(s)$ gives a structural guarantee that reward shaping
        or post-hoc filtering alone cannot match.
    \end{itemize}
  \end{exampleblock}
